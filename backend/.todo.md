# Marketly Backend TODO

This file tracks the next round of fixes and improvements for the scoring pipeline.

## 1. Fix missing fundamentals at the source

The current scoring layer is mostly fine. The bigger issue is missing upstream data, which causes many computed metrics to be `null`.

### What is missing now

- Income statement fields often missing from the stored payload:
  - `revenue`
  - `netIncome`
  - `operatingIncome`
  - `ebitda`
  - `costOfRevenue`
  - `eps`
- Balance-sheet fields are not fetched at all:
  - `totalDebt`
  - `totalAssets`
  - `totalStockholdersEquity`
- Interest-expense data is not fetched reliably:
  - needed for `interestCoverage`
- Quote-level fallback fields are not fetched:
  - `forwardEps`
  - `dividendRate`

### Required changes

- Relax the FMP income statement gate in [`app/integrations/financials.py`](/Users/artush/Developer/Marketly/backend/app/integrations/financials.py)
  - Right now income statements are only stored if `len(income) >= 4`
  - That is too strict
  - Store whatever valid periods are returned
- Add balance-sheet fetch and persist it in the merged payload
- Add cash-flow fetch and persist it in the merged payload
- Extend the typed model in [`app/models.py`](/Users/artush/Developer/Marketly/backend/app/models.py) for new financial blocks and quote fields

## 3. Make macroeconomics actually influence the score

Right now macro data is fetched and passed through, but it is not being handled in a structured enough way to reliably influence the final score.

### Problem

- Raw macro series are sent to GPT
- There is no deterministic summary of what the macro regime currently looks like
- GPT can ignore or underweight that block

### Required changes

- Add a macro summarizer service
  - produce a compact block like:
    - inflation trend
    - rates trend
    - unemployment trend
    - GDP trend
    - equity-market trend
    - oil / yield pressure flags
- Keep it simple and deterministic
- Pass that summary to GPT instead of only long raw time series
- Consider adding `macroSummary` to the `/score` response

## 4. Add micro / sector / company-context signals

The current pipeline uses company news and some fundamentals, but it does not build a proper micro-context layer.

### Missing now

### Required changes

- Add a micro-context block in scoring input:
  - sector
  - industry
  - competitive position clues
  - analyst recommendation trend
  - recent event flags
- Normalize recent company news into categories:
  - earnings
  - guidance
  - product launches
  - regulation / legal
  - layoffs / restructuring
  - M&A
  - analyst upgrades / downgrades

## 5. Make news materially affect the score

News is currently passed through, but the effect is too loose and too model-dependent.

### Problem

- GPT gets article payloads
- there is no intermediate structured sentiment / event summary
- recent news can be ignored or overweighted inconsistently

### Required changes

- Add a deterministic news summarizer before GPT
- Extract:
  - positive catalysts
  - negative catalysts
  - recurring themes
  - sentiment skew
  - major one-off risks
- Limit news to recent and relevant articles
- Prefer category summaries over raw article blobs in the final GPT payload

## 6. Rework the final scoring architecture

Right now the final score is still mostly GPT-led.

That is okay for explanations, but weaker for consistency.

### Better target design

Split the result into:
- deterministic subscores
  - profitability
  - growth
  - valuation
  - balance-sheet / stability
  - news / sentiment
  - macro regime
- GPT explanation layer
  - summary
  - positives
  - negatives
  - optional final adjustment within a narrow band

### Required changes

- Build actual numeric subscores per category
- Combine them with explicit weights
- Use GPT more for interpretation than for raw scoring
- Optionally let GPT adjust the deterministic score by a small bounded amount, not replace it
